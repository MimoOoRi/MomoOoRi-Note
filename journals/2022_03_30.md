- # async与await
- 概念
	- ES7(2016)新增的关键字，简化回调地狱的解决方案
	- ## `async`
		- 用于修饰函数，该函数为异步函数
		- 异步函数中才能使用`await`关键字
			- 如果`async`函数有返回值，会自动生成Promise对象，将返回值作为`resolve`的参数传入
	- ## `await`
		- ### 异步成功
			- 可以实现等待阻塞程序，等到当前Promise执行有结果(`resolve`或`reject`)后，将传入`resolve`或`reject`参数作为`await`的返回值返回，再继续执行，`async`函数剩下的内容
		- ### 异步失败`reject`
			- 利用try、catch捕获异步异常，来统一处理异步失败操作

	- 客户端/服务器端

	- browser/server

	- 将程序全部放在服务器上，用户通过浏览器访问服务器资源

	- 安装了服务器软件的计算机
- ## 前端与后端
	- 前端
		- 提供一个用户体验良好的应用程序
		- 主要实现用户交互和数据展示
	- 后端
		- 数据业务处理，接受前端数据，逻辑判断，操作数据库数据，将数据库数据发送给前端
		-
- ## URL
	- Uniform Resource Locator(统一资源定位器/全球资源定位符)
	- 用于定位互联网中的资源(资源的地址)
	- 格式：
		- `协议://主机:端口号/资源路径？参数`
		- 协议：
			- 一般为`http`或`https`
		- 主机
			- IP：
				- IPv4：0.0.0.0 - 255.255.255.255
				- IPv6：保证每台互联网上的计算机都有唯一的IP地址
			- 域名
				- `www.主机名称.com`
		- 端口号
			- 定位服务程序
			- 取值：0-65535
			- 申请端口尽量避免`0-1024`的端口，因为该范围的端口大部分被系统占用

- # HTTP协议
- 概念
	- 计算机网络协议
		- 计算机间通讯要遵循约定、规则
	- ## HTTP
		- hypertext transfer protocol 超文本传输协议
		- 每个请求只能请求服务器上的一个资源，服务器会对请求生成对应响应
	- ## 请求
		- 请求格式
			- 请求头(浏览器要告诉服务器的通用信息)
				- 请求首行
					- `请求类型 资源路径 HTTP协议版本`
			- 请求正文(GET没有请求正文，浏览器向服务器提交的数据)
				- 以键值对形式提交
		- 请求类型
			- `GET`
				- 向服务器索取资源的请求 authorization
			- `POST`
				- 向服务器提交数据的请求
	- ## 响应
		- 组成
		- 响应头(服务器要告诉浏览器的一些通用信息)
			- 响应首行
				- `HTTP版本 响应状态码`
		- 响应正文
	- ## 状态码
		- HTTP响应状态码，由3位数字组成
		- 常见状态码
			- 200：成功
			- 404：资源找不到
			- 500：服务器错误，(后端程序执行出错，抛出异常)
	- ## 缓存
		- 从服务器获取不常改变的资源后，存储到浏览器本地文件夹中，下次要使用资源时，先从本地文件查找，如果有，则直接使用，从而减少浏览器请求次数以及服务器响应次数，提高用户体验，减少流量消耗，减少服务器压力。
	- ## 特点
	  id:: 624402ff-5a5e-421b-b5ab-408ae1a31e36
		- 简单快速
			- 使用简单，仅需提供请求资源路径和方式，则可以发送HTTP请求
			- 传输速度快
		- 灵活
			- 支持各种资源的传输
		- 无连接
			- 短连接
				- 发送请求与响应前，先建立链接(建立通信通道)，将请求数据包发送到服务器，服务器生成响应数据包后，发送给浏览器主机，则该通道被注销
			- 长连接
				- 发送请求与响应前，先建立连接(建立通信通道),将请求数据包发送到服务器，服务器生成响应数据包后，发送给浏览器主机，该通道==不会马上被注销==〔一般会限制使用时间或请求总数）
			- 无状态
				- 不具备保存用户状态信息的能力
				- 比如：登录状态
		- 明文传输
			- 发送的数据不做加密处理
	- HTTP 与 HTPPS区别 #card
	  id:: 62440556-c0d7-4909-be71-ed132bef6396
		- 端口号
			- HTTP默认端口号为80，HTTPS端口号为443
		- 数据传输
			- HTTP数传输以明文形式传输，HTTPS数据以加密形式传输
		- 服务器身份认证
			- HTTP不具备识别目标服务器是否正规合法能力，HTTPS识别服务器是否正规
		- 响应速度
			- HTTP协议响应速度高于HTTPS，HTTPS(HTTP+SSL+TLS)
- # Nodejs
- 概述
  id:: 62440b9f-60bf-40ca-bb24-206072f88c10
	- 可以独立执行js的服务器环境
	- 底层基于谷歌的V8引擎实现
	- 可以使用Nodejs环境实现后端程序编写
- ### 执行js文件
	- 准备要执行的js文件
	- 执行目标js文件
		- 基于系统命令行执行
			- 在要执行的文件所在目录下，文件路径输入栏中输入cmd，打开命令行窗口
			- 命令行窗口输入`node 目标文件.js` 执行目标文件
		- 基于VSCode执行
			- 选中要执行的js文件，右键选择`在集成终端中打开`
			- 命令行窗口输入`node 目标文件.js` 执行目标文件
	- ==注意==
		- nodejs只支持ECMAScript执行，不支持BOM和DOM
- # npm
- ## 背景
	- 实际开发中，经常使用第三方库时，存在以下问题
		- 需要去各种网站下载对应包
		- 项目整合时，存在版本管理不方便
- ## 概念
	- npm(node package management, node包管理工具）：nodejs自带的命令，可以实现项目包的统一管理（包下载、删除、查看)。
- ## 操作流程
	- 创建项目文件夹
	- 初始化项目
	  id:: 62441b4c-d54e-4083-abbd-2ec21bb43898
		- 选中项目文件夹，右键选择`在集成终端中打开`，打开命令行窗口
		- 输入`npm init -y`并回车
			- `-y`：可选，一律认为`yes`
		- 下载目标包
			- 输入命令：`npm install 包名`
			- 注意包名全小写
	- ### 文件解释
		- `node_modules`：该文件夹用于存放使用`npm`命令下载的第三方包文件，每个包会以一个文件夹的形式存在
		- `package-lock.json`(不要修改)：记录依赖包信息，后期使用git上传项目时，用于版本控制
		- `package.json`：用于实时记录当前项目的依赖信息
			- 每个包会以`dependencies`或`DevDependencies`中一条记录的形式存在
	- ### npm常用命令
		- 查看包的版本信息
			- 查看 `npm view 包名 version`
			- 下载默认版本(最新) `npm install 包名@版本号`
				- 简写 `npm i 包名@版本号`
			- 卸载(删除)  `npm uninstall 包名`
				- 简写 `npm uni 包名`
			- 多文件操作
				- 下载`npm i 包名1 包名2...`
				- 卸载 `npm uni 包名1 包名2`
			- 下载`package.json`中配置的第三方依赖包`npm i`
			- ### 镜像地址
			- 默认镜像地址 `https://registry.npmjs.org`
			- 查看 `npm config get registry`
				- 修改 `npm config set registry https://registry.npm.taobao.org新的镜像地址`
				- 国内镜像地址 `https://registry.npm.taobao.org`
